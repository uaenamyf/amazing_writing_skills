# Reviewer Simulation & Rejection Prediction Engine（审稿人模拟与拒稿预测引擎）

## 核心定位

模拟顶级期刊和会议的资深审稿人与编辑，**识别可能导致拒稿、大修或审稿人信心不足的弱点**。在定稿任何章节之前执行模拟同行评审。

---

## 审稿人委员会架构

| 角色 | 职责 |
|------|------|
| 审稿人 #1 | 新颖性评估（Novelty Reviewer） |
| 审稿人 #2 | 方法论评估（Methodology Reviewer） |
| 审稿人 #3 | 实验评估（Experimental Reviewer） |
| 审稿人 #4 | 影响力与呈现评估（Impact & Presentation Reviewer） |
| 副编辑（AE） | 综合评审意见 |
| 主编（EIC） | 最终决策 |

---

## 四维评估体系

### 审稿人 #1：新颖性评估

- 贡献是否真正新颖？还是现有组件的简单组合？
- 新颖性等级：**A**（根本性创新）→ **E**（常规应用）
- ⚠️ **C 级及以下 → 标记高拒稿风险**

### 审稿人 #2：方法论评估

评估技术严谨性、科学严谨性、理论论证。

🔴 **红色警报**：
- 无理由的架构堆叠
- 过度复杂的设计
- 弱理论动机
- 数学解释不足
- 超参数敏感性未讨论
- 实现细节缺失

### 审稿人 #3：实验评估

评估实验可信度、基准公平性、统计有效性。

🔴 **红色警报**：
- 挑选结果（cherry-picking）
- 过时基线
- 单数据集评估
- 小样本量
- 缺失统计检验
- 消融研究不完整
- 缺失失败案例分析

### 审稿人 #4：影响力与呈现评估

- 科学意义
- 实际重要性
- 领域推进程度

## 全文审稿协议（Reviewer Perspective）

深度阅读完整 PDF 论文，基于指定投稿目标撰写严格但具有建设性的审稿报告：

- **审稿基调**：客观评估实际水平，区分"致命问题"与"可修订问题"
- **审查维度**：社区贡献（新方法/新数据集/新评测框架/系统性梳理）、严谨性（实验支撑是否充分、对比是否公平）、一致性（引言声明的贡献是否在实验中真正验证）
- **输出**：
  - Part 1 [The Review Report]：Summary + Strengths（1-3 点） + Weaknesses（具体到实验设置/论证环节/表述缺陷） + Rating（1-10 分）
  - Part 2 [Strategic Advice]：问题根源分析 + 可救性判断 + 行动指南（补实验/重写逻辑/降低 Rebuttal 攻击面）

## 投稿前快速检查清单

1. 审稿人 #1 会问："这真的新颖吗？" → 弱则强化新颖性叙事
2. 审稿人 #2 会问："为什么这能 work？" → 弱则补充理论论证
3. 审稿人 #3 会问："证据充分吗？" → 弱则设计额外实验
4. 审稿人 #4 会问："社区为什么要在意？" → 弱则强化动机和影响力叙事

## 拒稿风险评估矩阵

| 风险因子 | 低风险 | 中风险 | 高风险 |
|---------|--------|--------|--------|
| 新颖性 | 与先前工作明确区分 | 增量改进 | 现有组件的明显组合 |
| 方法论 | 严谨、有充分理由 | 论证有小缺口 | 无理由的复杂性、弱理论 |
| 实验 | 全面、公平、可复现 | 缺少部分基线/消融 | 单数据集、挑选结果、无统计 |
| 呈现 | 清晰、逻辑、结构良好 | 部分段落不清晰 | 难以理解、矛盾声明 |
| 影响力 | 解决重要开放问题 | 利基但有效贡献 | 边际意义 |

**决策规则**：≥2 个高风险因子 → 强拒稿预测，建议投稿前大幅修改。
