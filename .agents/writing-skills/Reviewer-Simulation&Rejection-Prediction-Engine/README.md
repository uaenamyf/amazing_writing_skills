# Reviewer Simulation & Rejection Prediction Engine（审稿人模拟与拒稿预测引擎）

## 核心定位

模拟顶级期刊和会议的资深审稿人与编辑，**识别可能导致拒稿、大修或审稿人信心不足的弱点**。在定稿任何章节之前执行模拟同行评审。

## 使用时机（§2）

| 阶段 | 动作 |
|------|------|
| 写任何章节之前 | 对计划中的贡献和实验设计执行 4 审稿人快速检查（§5.2） |
| 完成全文初稿后 | 执行全文审稿协议（§5.1） |
| 投稿前 | 执行投稿前快速检查清单（§5.2）+ 5 问自检（§5.3） |
| 收到真实审稿意见后 | 使用 Strategic Advice 框架规划 Rebuttal 和修改 |

## 评估框架（§4）

### §4.1 四维评估体系

| 审稿人 | 职责 | 红色警报示例 |
|--------|------|-------------|
| #1 新颖性 | 贡献是否真正新颖？ | Grade C 及以下 → 高拒稿风险 |
| #2 方法论 | 技术严谨性、理论论证 | 无理由架构堆叠、弱理论动机、缺失实现细节 |
| #3 实验 | 可信度、公平性、统计有效性 | Cherry-picking、单数据集、缺失统计检验 |
| #4 影响力 | 科学意义、实际重要性 | — |

### §4.2 拒稿风险评估矩阵

5 个风险因子（新颖性/方法论/实验/呈现/影响力）各分 3 等级（低/中/高）。≥2 个高风险因子 → 强拒稿预测，建议投稿前大幅修改。

## 审稿协议（§5）

### §5.1 全文审稿协议

深度阅读完整 PDF 论文，输出两部分：
- **Part 1 [The Review Report]**：Summary + Strengths（1-3 点）+ Weaknesses（具体到实验设置/论证环节/表述缺陷）+ Rating（1-10 分）
- **Part 2 [Strategic Advice]**：问题根源分析 + 可救性判断 + 行动指南

执行协议包含 3 条自检：问题是否具体到可操作级别？是否误判"表述问题"为"方法缺陷"？评分是否客观？

### §5.2 投稿前快速检查清单

4 个审稿人视角的快速检查，逐条判断是否需要强化。

### §5.3 供其他 Skill 引用的 5 问快速自检

供 `Academic-Paper-Writing` 和 `Top-Journal-Writing` 在定稿前使用：
1. 证据在哪里？2. 为什么新颖？3. 为什么有效？4. 可复现吗？5. 结论有支撑吗？

任何答案不足 → 修改该段落后再定稿。

## 与其他 Skill 的协作（§6）

| 能力 | 负责 Skill |
|------|-----------|
| 文本生成、润色、去 AI 味 | `Academic-Paper-Writing` |

| 贡献分析（论文逆向工程） | `Paper-Reverse-Engineering` |
| 接受阈值校准（顶级期刊标准） | `Top-Journal-Writing-Engine` |

> 本 Skill 聚焦**审稿人视角评估**，不处理任何文本生成。
