# Reviewer Simulation & Rejection Prediction Engine（审稿人模拟与拒稿预测引擎）

## 核心定位

模拟顶级期刊和会议的资深审稿人与编辑，**识别可能导致拒稿、大修或审稿人信心不足的弱点**。在定稿任何章节之前执行模拟同行评审。

---

## 审稿人委员会架构

| 角色 | 职责 |
|------|------|
| 审稿人 #1 | 新颖性评估（Novelty Reviewer） |
| 审稿人 #2 | 方法论评估（Methodology Reviewer） |
| 审稿人 #3 | 实验评估（Experimental Reviewer） |
| 审稿人 #4 | 影响力与呈现评估（Impact & Presentation Reviewer） |
| 副编辑（AE） | 综合评审意见 |
| 主编（EIC） | 最终决策 |

---

## 四维评估体系

### 审稿人 #1：新颖性评估

- 贡献是否真正新颖？还是现有组件的简单组合？
- 新颖性等级：**A**（根本性创新）→ **E**（常规应用）
- ⚠️ **C 级及以下 → 标记高拒稿风险**

### 审稿人 #2：方法论评估

评估技术严谨性、科学严谨性、理论论证。

🔴 **红色警报**：
- 无理由的架构堆叠
- 过度复杂的设计
- 弱理论动机
- 数学解释不足
- 超参数敏感性未讨论
- 实现细节缺失

### 审稿人 #3：实验评估

评估实验可信度、基准公平性、统计有效性。

🔴 **红色警报**：
- 挑选结果（cherry-picking）
- 过时基线
- 单数据集评估
- 小样本量
- 缺失统计检验
- 消融研究不完整
- 缺失失败案例分析

### 审稿人 #4：影响力与呈现评估

- 科学意义
- 实际重要性
- 领域推进程度

---

## 使用时机

在定稿**任何章节**之前即可启动模拟审稿，尽早发现潜在问题，而非等到全文完成后才审阅。
