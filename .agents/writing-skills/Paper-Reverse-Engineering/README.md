# Paper Reverse Engineering（论文逆向工程）

## 核心定位

当提供目标论文时，**不直接模仿措辞**，而是逆向工程其底层学术写作策略，并使用提取的风格模式重构新内容。

## 与 Author-Style-Fingerprint 的区别（§1）

| 维度 | Paper Reverse Engineering | Author Style Fingerprint |
|------|--------------------------|--------------------------|
| 输入 | 单篇目标论文 | 同一作者/研究组的多篇论文（3-20 篇） |
| 输出 | 论文写作策略蓝图 | 作者级风格指纹模型 |
| 用途 | 理解某篇论文为什么这样写 | 学习某位作者如何思考和论证 |

共用模块（贡献分类、语言风格画像、期刊指纹、反抄袭约束、融合模式、输出模式）由 `Author Style Fingerprint Engine` 和 `Top-Journal Writing Engine` 统一维护。

## 逆向工程流水线（§2）

### §2.1 结构分解
识别论文类型（方法/应用/基准/综述/理论等），构建论文蓝图：研究问题 → 核心假设 → 科学动机 → 贡献策略 → 验证策略。

### §2.2 引言逻辑提取
逐层分析动机构建：背景层 → 当前进展 → 现有局限 → 知识空白 → 研究问题 → 提出方案 → 贡献。识别 3 种引言模式（Pattern A/B/C），**复用模式而不复用措辞**。

### §2.3 贡献分析
识别目标论文的实际新颖之处。贡献类型分类（8 种）引用 `Author-Style-Fingerprint-Engine` §3.3。提取新颖性表达方式，生成等效替代。

### §2.4 方法章节逆向
提取：模块组织 → 信息流 → 命名惯例 → 公式引入风格 → 伪代码风格 → 消融逻辑。复用架构语法，不复用架构名称。

### §2.5 实验策略挖掘
逆向分析实验组织顺序：数据集 → 指标 → 实现 → 对比 → 消融 → 可视化 → 讨论 → 失败案例。

### §2.6 结果分析逆向
提取每个图表/表格的：观察模式 → 解释模式 → 结论模式。复用分析链，避免措辞重叠。

### §2.7 图表标题风格
分析标题长度、信息密度、引用风格、统计报告风格。

### §2.8 讨论风格建模
识别局限性讨论方式（温和 / 均衡 / 激进自我批评），提取未来工作逻辑和改进路径。

## 质量保证（§3）

### §3.1 审稿人仿真层
生成后评估：审稿人能识别出复制文本吗？新颖性有理由吗？证据支持结论吗？能通过相似度检测吗？

### §3.2 优先级
逻辑 > 结构 > 论证 > 语言。永远不把措辞模仿置于科学推理模仿之上。
